# 该项目网络同步相关的且从零开始构建的新手教程（UE5.7 / PuzzlePlatforms）

> 适用人群：
> 1) 第一次做 UE 多人联机
> 2) C++ 语法还不熟
> 3) 看代码容易晕，需要一步一步照着做

---

## 一、先说人话：你到底要做成什么

你最终要看到的是：

1. 开两个游戏窗口（或两台电脑）
2. A 点 `Host` 建房
3. B 点 `Join` 找到房间并加入
4. 两人进同一个 Lobby（大厅）
5. 到人数后自动切到 Game 地图
6. 地图里的移动平台在两边看起来一致（这就是网络同步）

一句话总结：
**菜单发起联机 -> GameInstance 管 Session -> 服务器计算并同步世界状态。**

## 三、先懂3个核心词（不懂就会卡）

## 1) Session（会话）
可以理解成“房间”。
- `CreateSession` = 开房
- `FindSessions` = 找房
- `JoinSession` = 进房

## 2) 服务器权威（Authority）
多人游戏里，谁说了算？
答案：**服务器**。

- 服务器：负责“算结果”（比如平台现在在哪）
- 客户端：负责“显示结果”（看到平台在哪）

## 3) Replication（复制/同步）
把服务器上的 Actor 状态复制给客户端。

- `SetReplicates(true)`：这个 Actor 允许被同步
- `SetReplicateMovement(true)`：这个 Actor 的移动会被同步

## 四、先把工程配置对（从0开始最重要）

这里按本项目现状来，照抄即可。

## 1) 引擎与插件
本项目 `PuzzlePlatforms.uproject` 已是 UE5.7，并启用 `OnlineSubsystemSteam` 插件。

## 2) Build.cs 模块依赖
文件：`Source/PuzzlePlatforms/PuzzlePlatforms.Build.cs`

必须有：
- `OnlineSubsystem`
- `OnlineSubsystemUtils`
- 动态加载 `OnlineSubsystemSteam`

否则现象：编译报 Online 相关类型找不到，或者运行时拿不到 Session 接口。

## 3) DefaultEngine.ini 的关键项
文件：`Config/DefaultEngine.ini`

关键项：

```ini
[/Script/EngineSettings.GameMapsSettings]
GameInstanceClass=/Script/PuzzlePlatforms.PuzzlePlatformsGameInstance

[/Script/Engine.GameEngine]
+NetDriverDefinitions=(DefName="GameNetDriver",DriverClassName="OnlineSubsystemSteam.SteamNetDriver",DriverClassNameFallback="OnlineSubsystemUtils.IpNetDriver")

[OnlineSubsystem]
DefaultPlatformService=Steam

[OnlineSubsystemSteam]
bEnabled=False
SteamDevAppId=480
```

### 为什么现在是 `bEnabled=False`？
因为本机双开调试更稳定。

- 本地调流程：`bEnabled=False` 或加 `-nosteam`
- 真 Steam 联机：再改 `bEnabled=True`，用两账号多机测试

## 六、从零开始搭建：按这个顺序做

## Step 1：先让菜单能“命令”联机系统

你在 UI 按钮里不要直接写底层 Session 调用，应该通过接口转发。

文件：`MenuInterface.h`

定义这些函数：
- `Host(FString ServerName)`
- `Join(uint32 Index)`
- `RefreshServerList()`
- `LoadMainMenu()`

然后由 `UPuzzlePlatformsGameInstance` 去实现它。

这样做的好处：
- UI 不懂网络细节，只负责“点击触发”
- 联机逻辑集中在 GameInstance，清晰不乱

---

## Step 2：GameInstance 初始化 OnlineSubsystem

文件：`PuzzlePlatformsGameInstance.cpp` -> `Init()`

要做的事：

1. `IOnlineSubsystem::Get()` 拿子系统
2. `GetSessionInterface()` 拿 Session 接口
3. 绑定4个异步回调：
- `OnCreateSessionComplete`
- `OnDestroySessionComplete`
- `OnFindSessionsComplete`
- `OnJoinSessionComplete`

### 新手理解
这些回调就像“快递到达通知”：
你发了请求（开房/找房），它不会立刻返回，要等系统完成后再回调你。

---

## Step 3：实现 Host（建房）

入口：`MainMenu.cpp` 的 `HostServer()`

流程：
1. 读输入框房间名
2. 调 `MenuInterface->Host(ServerName)`
3. GameInstance 里 `Host()` 判断是否已有旧会话
4. 有旧会话先 `DestroySession`，回调里再 `CreateSession`
5. 没旧会话直接 `CreateSession`

`CreateSession()` 里关键参数：
- `bShouldAdvertise = true`（房间可被发现）
- `bUsesPresence = true`
- `bUseLobbiesIfAvailable = true`
- `NumPublicConnections = 5`
- 自定义 `ServerName`

创建成功后：
- `ServerTravel("/Game/PuzzlePlatforms/Maps/Lobby?listen")`

### `?listen` 是什么
表示当前主机也当服务器监听连接。
不加它，别人常常进不来。

---

## Step 4：实现 Join（搜房+进房）

入口：`MainMenu.cpp` 的 `OpenJoinMenu()` 和 `JoinServer()`

流程：
1. 打开 Join 面板时调用 `RefreshServerList()`
2. GameInstance 创建 `FOnlineSessionSearch`
3. 设置 `SEARCH_LOBBIES=true`
4. 调 `FindSessions`
5. 在 `OnFindSessionsComplete` 里把结果转成 UI 列表数据
6. 玩家选中某一行后，调用 `Join(Index)`
7. `OnJoinSessionComplete` 成功后拿连接地址
8. `ClientTravel(Address, TRAVEL_Absolute)` 跳过去

### 为什么有时候“找得到但进不去”
常见是这三项不匹配：
- 创建时 Lobby/Presence 设置
- 搜索时 Query 设置
- Steam 当前状态（本机双开冲突）

---

## Step 5：Lobby 自动开局

文件：`LobbyGameMode.cpp`

逻辑：
1. `PostLogin`：每来一个玩家就 `++NumberOfPlayers`
2. 玩家数达到阈值（项目里是2）就开定时器
3. `StartGame()`：
- `GameInstance->StartSession()`
- `bUseSeamlessTravel = true`
- `ServerTravel("/Game/PuzzlePlatforms/Maps/Game?listen")`

### 为什么要 SeamlessTravel
切图时连接更平滑，玩家状态处理更稳定。

---

## Step 6：网络同步重头戏（移动平台）

文件：`MovingPlatform.cpp`

这是项目里最值得学的同步范例。

### 正确做法（项目已采用）

1. 构造函数：
- `SetReplicates(true)`

2. `BeginPlay`：
- 仅服务器 `HasAuthority()` 内 `SetReplicateMovement(true)`

3. `Tick`：
- 仅服务器计算移动并 `SetActorLocation`
- 客户端不算，只接收复制结果

### 结果
所有客户端看到同一平台位置，避免“我这边在左边，你那边在右边”。

## 七、UE 编辑器操作（照点不容易错）

参考 `UEeditor.md`，按下面操作。

## 1) 角色输入（避免进游戏不能动）
1. 建 `IMC_Default`
2. 建 `IA_Move / IA_Look / IA_Jump`
3. 在角色蓝图赋值 `DefaultMappingContext` 和3个 Action

## 2) 主菜单控件命名（避免按钮失效）
`WBP_MainMenu` 中控件名要和 C++ `BindWidget` 一致：
- `HostButton` `JoinButton` `QuitButton`
- `CancelJoinMenuButton` `ConfirmJoinMenuButton`
- `CancelHostMenuButton` `ConfirmHostMenuButton`
- `MenuSwitcher` `MainMenu` `JoinMenu` `HostMenu`
- `ServerHostName` `ServerList`

## 3) Lobby 地图 GameMode 覆盖
在 Lobby 地图的 `World Settings` 里设 `GameMode Override = LobbyGameMode`。

## 4) 移动平台摆放
1. 场景放 `BP_MovingPlatform`
2. 设置 `Speed` 与 `TargetLocation`
3. 确认平台按 C++ 配置参与复制

## 5) 触发器绑定平台
1. 场景放 `BP_PlatformTrigger`
2. 在 `PlatformsToTrigger` 数组中拖入目标平台

## 八、零基础语法小抄（只保留本项目最常见）

- `if (!ensure(X)) return;`
  - 运行时检查，X不成立就提前退出，防止空指针崩溃

- `TSharedPtr<FOnlineSessionSearch>`
  - 智能指针，自动管理搜索对象生命周期

- `AddUObject(this, &Class::Func)`
  - 绑定成员函数回调

- `Cast<类型>(对象)`
  - 安全类型转换，失败返回 `nullptr`

- `HasAuthority()`
  - 判断当前是不是服务器权威端

## 九、对照组：常见错误 vs 正确写法

| 场景 | ❌ 错误 | ✅ 正确 |
|---|---|---|
| Actor 复制开关 | `SetReplicates(true)` 写在 `BeginPlay + HasAuthority` | `SetReplicates(true)` 写在构造函数 |
| 平台移动 | 客户端也在 Tick 里算位置 | 只在服务器 Tick 算，客户端接收同步 |
| 切图 | `ServerTravel("/Map/Lobby")` | `ServerTravel("/Map/Lobby?listen")` |
| UI 架构 | 按钮直接调 `SessionInterface` | 按钮 -> `IMenuInterface` -> `GameInstance` |
| Steam 测试 | 一开始就 `bEnabled=True` 本机双开 | 先本机 `bEnabled=False/-nosteam` 跑通流程 |
| UE5 头文件 | `#include "OnlineSessionInterface.h"` | `#include "Interfaces/OnlineSessionInterface.h"` |

## 十一、排错速查（出现问题直接对号入座）

## 问题1：看不到服务器列表
检查：
1. 是否调用了 `RefreshServerList()`
2. 搜索是否设置 `SEARCH_LOBBIES=true`
3. Host 端会话是否 `bShouldAdvertise=true`
4. Steam 是否冲突（本机双开常见）

## 问题2：能进房，但平台不同步
检查：
1. `AMovingPlatform` 构造函数是否 `SetReplicates(true)`
2. 服务器端是否 `SetReplicateMovement(true)`
3. `Tick` 是否被 `HasAuthority()` 限制在服务器执行

## 问题3：Host 成功，别人连不上
检查：
1. `ServerTravel` 是否带 `?listen`
2. `DefaultEngine.ini` 的 `NetDriverDefinitions` 是否正确
3. 是否确实进入了 Lobby 地图而不是停留在菜单图

## 十三、1分钟总复习

你只要记住5句：

1. 联机核心写在 `GameInstance`，不是写在 Widget。
2. UI 只负责按钮，调用接口，不直接碰 Session 细节。
3. 多人世界里服务器说了算，客户端不要自作主张算结果。
4. 复制开关位置很关键：`SetReplicates(true)` 放构造函数。
5. 调试顺序是：本机流程跑通 -> 再做 Steam 真实联机。

到这里，你已经能从零搭起这个项目的网络同步主干了。
